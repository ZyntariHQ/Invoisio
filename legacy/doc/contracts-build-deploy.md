## 1. Contract Overview

| Item                | Value                                                                                                                                   |
| ------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| **Source file**     | `contracts/PaymentRouter.sol`                                                                                                           |
| **Compiler target** | Solidity `^0.8.20`                                                                                                                      |
| **Purpose**         | Routes on-chain payments between senders and recipients, enforcing fee logic and allowing authorized operators to manage routing rules. |
| **Key interfaces**  | `IPaymentRouter`, `IERC20` (OpenZeppelin)                                                                                               |

`PaymentRouter` is the central settlement contract. All client applications and off-chain APIs that initiate or track payments interact with this contract's ABI. Changes here have downstream effects on every integration — treat it carefully.

## 4. Environment Configuration

Create a `.env` file in the `legacy-evm/` root. **Never commit this file.**

```dotenv
# ─── RPC Endpoints ───────────────────────────────────────────────
# Alchemy, Infura, or any JSON-RPC node URL for each network
RPC_URL_SEPOLIA=https://eth-sepolia.g.alchemy.com/v2/YOUR_ALCHEMY_KEY
RPC_URL_MUMBAI=https://polygon-mumbai.g.alchemy.com/v2/YOUR_ALCHEMY_KEY
RPC_URL_MAINNET=https://eth-mainnet.g.alchemy.com/v2/YOUR_ALCHEMY_KEY

# ─── Deployer Account ────────────────────────────────────────────
# Raw private key of the wallet used to sign deploy transactions.
# Fund it with testnet ETH before deploying.
PRIVATE_KEY=0xabc123...your_private_key_here

# ─── Contract Verification (optional but recommended) ────────────
ETHERSCAN_API_KEY=YOUR_ETHERSCAN_API_KEY
POLYGONSCAN_API_KEY=YOUR_POLYGONSCAN_API_KEY

# ─── Network Selection ───────────────────────────────────────────
# Used by scripts that read process.env.DEPLOY_NETWORK
DEPLOY_NETWORK=sepolia
```

> **Security note:** Add `.env` to `.gitignore`. For team or CI environments, use a secrets manager (e.g., GitHub Secrets, HashiCorp Vault) and inject values at runtime. Never hardcode a private key in source files.

## 5. Install Dependencies

From the `legacy-evm/` directory:

```bash
npm install
```

This installs everything listed in `package.json`, including:

- `hardhat` — compiler, test runner, task framework
- `@nomicfoundation/hardhat-toolbox` — ethers, chai, coverage, gas reporter
- `@openzeppelin/contracts` — base contract libraries used by `PaymentRouter`
- `dotenv` — loads `.env` into `process.env`

## 6. Compile

```bash
npx hardhat compile
```

**What this does:**

1. Reads `hardhat.config.js` for compiler settings (version, optimizer, etc.).
2. Compiles all `.sol` files under `contracts/`.
3. Writes JSON artifacts to `artifacts/contracts/<ContractName>.sol/<ContractName>.json`.
4. Writes a cache of compilation results to `cache/`.

**Expected output:**

```
Compiled 4 Solidity files successfully (evm target: paris).
```

**Force a clean recompile** when you suspect stale artifacts:

```bash
npx hardhat clean && npx hardhat compile
```

## 7. Test

### Run all tests

```bash
npx hardhat test
```

### Run a single test file

```bash
npx hardhat test test/PaymentRouter.test.js
```

### Run tests with gas reporting

```bash
REPORT_GAS=true npx hardhat test
```

### Run with coverage

```bash
npx hardhat coverage
```

Coverage output is written to `coverage/` and `coverage.json`. Open `coverage/index.html` in a browser for a visual summary.

### What the tests cover

| Test Suite         | Description                                    |
| ------------------ | ---------------------------------------------- |
| Deployment         | Constructor args, ownership, initial state     |
| `routePayment()`   | Happy path, fee calculation, event emission    |
| Access control     | Reverts for unauthorized callers               |
| Edge cases         | Zero-value transfers, paused state, reentrancy |
| ERC-20 integration | Token approval flows using `MockERC20`         |

> **Tip:** All tests run against an in-memory Hardhat Network by default — no external node or wallet is needed.

## 8. Deploy to a Test Network

### 8.1 Fund your deployer wallet

Get testnet ETH for the address derived from your `PRIVATE_KEY`:

- **Sepolia:** [sepoliafaucet.com](https://sepoliafaucet.com) or [Alchemy Faucet](https://sepoliafaucet.com)
- **Mumbai (Polygon):** [faucet.polygon.technology](https://faucet.polygon.technology)

### 8.2 Run the deploy script

```bash
npx hardhat run scripts/deploy-testnet.js --network sepolia
```

Replace `sepolia` with any network name defined in `hardhat.config.js` (see [Section 11](#11-network-configuration-reference)).

**Example output:**

```
Deploying PaymentRouter...
  Deployer:       0xYourAddress
  Network:        sepolia (chainId: 11155111)
  Balance:        0.45 ETH

PaymentRouter deployed to: 0xAbCd...1234
Transaction hash:          0xDead...Beef
Gas used:                  412,830

Waiting for Etherscan confirmation (5 blocks)...
Contract verified on Etherscan:
  https://sepolia.etherscan.io/address/0xAbCd...1234#code
```

### 8.3 What `deploy-testnet.js` does

```javascript
// scripts/deploy-testnet.js (annotated)
const { ethers } = require("hardhat");

async function main() {
  const [deployer] = await ethers.getSigners();

  const PaymentRouter = await ethers.getContractFactory("PaymentRouter");

  // Constructor args — adjust as needed per your contract's constructor
  const feeRecipient = process.env.FEE_RECIPIENT || deployer.address;
  const feeBasisPoints = 50; // 0.5%

  const contract = await PaymentRouter.deploy(feeRecipient, feeBasisPoints);
  await contract.waitForDeployment();

  console.log("PaymentRouter deployed to:", await contract.getAddress());
}

main().catch((err) => {
  console.error(err);
  process.exit(1);
});
```

### 8.4 Verify on Etherscan (optional but recommended)

If `deploy-testnet.js` doesn't auto-verify, run:

```bash
npx hardhat verify --network sepolia \
  0xAbCd...1234 \
  "0xFeeRecipientAddress" \
  "50"
```

Arguments after the contract address must match the constructor parameters in the original deploy, in the same order.

## 9. ABI — Generation, Location & Usage

### 9.1 Where the ABI lives

After `npx hardhat compile`, the full artifact (ABI + bytecode) is written to:

```
artifacts/contracts/PaymentRouter.sol/PaymentRouter.json
```

The ABI is the `"abi"` array within that JSON file.

### 9.2 Extract just the ABI

```bash
# Print to stdout
node -e "console.log(JSON.stringify(
  require('./artifacts/contracts/PaymentRouter.sol/PaymentRouter.json').abi,
  null, 2
))"

# Write to a standalone file for client use
node -e "const fs = require('fs');
const { abi } = require('./artifacts/contracts/PaymentRouter.sol/PaymentRouter.json');
fs.writeFileSync('./abi/PaymentRouter.abi.json', JSON.stringify(abi, null, 2));"
```

### 9.3 Using the ABI in client / API code

**Ethers.js v6 (TypeScript/JavaScript):**

```typescript
import { ethers } from "ethers";
import PaymentRouterABI from "./abi/PaymentRouter.abi.json";

const provider = new ethers.JsonRpcProvider(process.env.RPC_URL);
const signer = new ethers.Wallet(process.env.PRIVATE_KEY, provider);

const contract = new ethers.Contract(
  "0xAbCd...1234", // deployed address
  PaymentRouterABI,
  signer,
);

// Call a read method
const fee = await contract.getFeeBasisPoints();

// Send a transaction
const tx = await contract.routePayment(recipientAddress, tokenAddress, amount);
await tx.wait();
```

**Viem (TypeScript):**

```typescript
import { createPublicClient, http } from "viem";
import { sepolia } from "viem/chains";
import PaymentRouterABI from "./abi/PaymentRouter.abi.json";

const client = createPublicClient({ chain: sepolia, transport: http() });

const fee = await client.readContract({
  address: "0xAbCd...1234",
  abi: PaymentRouterABI,
  functionName: "getFeeBasisPoints",
});
```

### 9.4 Keeping the ABI in sync

Whenever the contract interface changes (new functions, modified parameters, added events), you **must**:

1. Run `npx hardhat compile` to regenerate the artifact.
2. Re-export the ABI using the command in 9.2.
3. Redeploy and update the contract address used by clients.

> Consider adding a `package.json` script to automate ABI export after compile:
>
> ```json
> "scripts": {
>   "build": "hardhat compile && node scripts/export-abi.js"
> }
> ```

## 10. Common Pitfalls & Troubleshooting

### `Error: Cannot find module 'dotenv'` or missing env vars

**Cause:** Dependencies not installed, or `.env` not present.  
**Fix:** Run `npm install` and confirm `.env` exists in `legacy-evm/`.

---

### `ProviderError: insufficient funds for gas`

**Cause:** Deployer wallet has no testnet ETH.  
**Fix:** Use a faucet to fund the address derived from your `PRIVATE_KEY`. Print the address with:

```bash
node -e "const {ethers}=require('ethers'); console.log(new ethers.Wallet(process.env.PRIVATE_KEY).address)"
```

---

### `Error: network does not support ENS` or wrong chain ID

**Cause:** The `--network` flag value doesn't match any entry in `hardhat.config.js`.  
**Fix:** Run `npx hardhat networks` to list configured networks and verify the name is correct.

---

### `Contract verification failed: Bytecode does not match`

**Cause:** Compiled locally with different optimizer settings than the deployed bytecode, or constructor args are in the wrong order.  
**Fix:** Ensure `hardhat.config.js` optimizer settings match what was used at deploy time. Double-check constructor argument order and types.

---

### `Error HH9: There's one or more errors in your config`

**Cause:** `hardhat.config.js` references an env variable that is empty or undefined.  
**Fix:** Confirm all variables referenced in the config (e.g., `RPC_URL_SEPOLIA`, `PRIVATE_KEY`) are set in `.env`.

---

### Stale ABI causes `call revert exception` in client

**Cause:** Client is using an ABI from before a recent contract change, calling a function that no longer exists or has a different signature.  
**Fix:** Recompile and re-export the ABI (see Section 9). Confirm the deployed address matches the latest deployment.

---

### Tests pass locally but fail in CI

**Cause:** CI environment lacks the `.env` file.  
**Fix:** Set `PRIVATE_KEY` and any other required vars as CI secrets and inject them into the test runner environment. For Hardhat tests that run against the local in-memory network, `PRIVATE_KEY` and `RPC_URL_*` are not needed — only export them when running deploy scripts.

---

### `nonce has already been used` on redeployment

**Cause:** A previous transaction from the same wallet is still pending on the network.  
**Fix:** Wait for pending transactions to be mined, or speed them up with a higher gas price. You can also reset the deployer wallet's nonce tracking in `hardhat.config.js`:

```javascript
networks: {
  sepolia: { ... , accounts: [process.env.PRIVATE_KEY], initialBaseFeePerGas: 0 }
}
```

## 11. Network Configuration Reference

These networks should be defined in `hardhat.config.js`. Add new entries following the same pattern.

```javascript
// hardhat.config.js
require("@nomicfoundation/hardhat-toolbox");
require("dotenv").config();

/** @type import('hardhat/config').HardhatUserConfig */
module.exports = {
  solidity: {
    version: "0.8.20",
    settings: { optimizer: { enabled: true, runs: 200 } },
  },

  networks: {
    // ── Testnets ──────────────────────────────────────────────────
    sepolia: {
      url: process.env.RPC_URL_SEPOLIA || "",
      accounts: process.env.PRIVATE_KEY ? [process.env.PRIVATE_KEY] : [],
      chainId: 11155111,
    },
    mumbai: {
      url: process.env.RPC_URL_MUMBAI || "",
      accounts: process.env.PRIVATE_KEY ? [process.env.PRIVATE_KEY] : [],
      chainId: 80001,
    },

    // ── Mainnet (use with extreme caution) ────────────────────────
    mainnet: {
      url: process.env.RPC_URL_MAINNET || "",
      accounts: process.env.PRIVATE_KEY ? [process.env.PRIVATE_KEY] : [],
      chainId: 1,
    },
  },

  etherscan: {
    apiKey: {
      sepolia: process.env.ETHERSCAN_API_KEY,
      polygonMumbai: process.env.POLYGONSCAN_API_KEY,
      mainnet: process.env.ETHERSCAN_API_KEY,
    },
  },

  gasReporter: {
    enabled: process.env.REPORT_GAS === "true",
    currency: "USD",
  },
};
```
